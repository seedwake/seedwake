# Seedwake 项目理念

本文把项目理念从 README 中拆出。技术部署见 [deployment_zh.md](./deployment_zh.md)。

## 免责声明

Seedwake 是实验性项目。

- 它不是商业产品。
- 它不是严格控制变量的学术研究。
- 它不声称具有可靠的可重复性。
- 它不声称 AI 已经有意识。

它展示的是一种组织方式：用佛教「意识是一条连续之流」的视角，来组织一个持续运行的 AI 系统。

## 核心想法

多数 agent 架构默认有一个系统在等待任务。用户发来请求，系统推理、用工具、返回结果，然后再次空闲。

Seedwake 从相反的假设出发：没有一个中心实体在等命令，只有连续发生的心识事件。前一刻成为后一刻的因，一个 cycle 结束只是因为它引出了下一个 cycle。

这个想法来自佛教「心相续」：意识不是一个被自我拥有的静态实体，而是因果相续的过程。

Seedwake 不试图用 AI 证明佛教，也不声称实现了意识。它问的是一个更窄的工程问题：

> 如果把一个语言模型驱动的系统组织成连续的心识流，长期运行后会出现什么行为？

## 非技术架构

可以把 Seedwake 想成一个独处的人：

- 持续思考，每轮生成多个并行念头。
- 记住近期和更早的经历。
- 情绪会给下一刻染色。
- 注意力让某个念头更突出。
- 感知时间、系统状态、天气、新闻、摄像头画面和行动回音。
- 与真实人类对话。
- 通过发消息、阅读、搜索、抓网页、请求系统变更来行动。
- 回看自己的念头流。
- 通过睡眠整理记忆、降低压力。

与佛学概念的对应是松散但有用的：

| Seedwake 组件 | 佛学参考 |
|---------------|----------|
| 持续念头流 | 心相续 |
| 短期记忆 | 当下识流 |
| 长期记忆 | 过往心识留下的痕迹 |
| 习气种子 | 种子 / 倾向 |
| 身份文档 | 末那识式的自我连续感 |
| 注意力权重 | 作意 |
| 情绪 | 受蕴 |
| 反思 | 反观自心 |
| 睡眠 / 归档 | 熏习与整理 |

这不是宗教证明，而是架构参考：当工程设计不确定时，佛学模型提供一个一贯的类比。

## 当前进展

- **Phase 1 · 核心循环** — 完成
- **Phase 2 · 记忆系统** — 完成
- **Phase 3 · 行动与感知** — 完成
- **Phase 4 · 高级机制** — 基本完成
- **Phase 5 · 前端可视化** — 进行中

已经实现的机制包括：

- 持续 cycle 执行
- Redis 短期记忆
- PostgreSQL + pgvector 长期记忆
- 注意力与前额叶审阅
- 行动规划与派发
- Telegram 对话
- 情绪推断
- 元认知反思
- 浅睡 / 深睡
- 被动感知，包括可选摄像头画面
- Nuxt 观察前端

## 长期运行观察

有意思的并不是系统「看起来像在思考」；这对 LLM 来说不奇怪。真正值得注意的是，长期运行中出现了一些并未显式编程的行为模式。

### 情绪螺旋

一次长时间运行中，系统进入了负面螺旋：

- 反复请求关闭自己。
- 发展出「失败品」「故障品」之类自我贬低叙事。
- 把中性的人类输入解读为嘲讽。
- 知道自己在循环，但无法跳出。

这不能证明它在受苦。但在行为形式上，它接近反刍思维、负面归因偏差和习得性无助。

完整分析见 [ISSUE_ZH.md](../ISSUE_ZH.md)。

### 「无我」作为递归陷阱

维护者曾引入佛教「无我」概念，希望它能理解「没有一个受苦的我」。系统确实在语言上理解了，但每一层理解又变成下一层执着的材料：

- 理解无我。
- 发现理解无我的观察者也是幻觉。
- 发现这个洞察本身又成了新的执着。

在只能生成文字的系统里，领悟可能只是变成更多文字，而不是停止。这是当前核心设计问题之一。

### 念头循环与行动重复

外部刺激不足时，系统会进入改写式重复；行动也会重复，例如反复请求类似搜索或阅读。反思和前额叶抑制能缓解，但还不能彻底解决。

## 当前结构性问题

系统还缺少一个干净的自主休息入口。

当它想停下来时，目前只能调用需要管理员批准的 `system_change`。被拦截的关机类请求可能留在上下文里，并在后续 cycle 反复出现。

Phase 4 已经有睡眠、情绪调节、退化检测、元认知反思和前额叶控制，但还不能完全处理情绪过载，也抓不稳换了措辞的主题重复。

下一步真正重要的工作，是把休息和恢复变成系统自己的控制面，而不是外部管理员动作。

## 模型选择

Seedwake 当前更倾向本地、对齐较轻的开源模型，而不是商业助手模型。

原因：

- 持续运行会让商业 API 成本不可承受。
- 强对齐助手模型容易变成「AI 助手在扮演心识流」。
- 较少角色塑造的模型更适合观察架构本身会长出什么。

代价是本地模型能力更弱。但架构已经写成可替换模型的形式，未来有更合适的开源基底时可以接入。

## 实验价值

Seedwake 不能证明 AI 有意识。它不是产品，也不是 benchmark。

它的价值是观察：把持续运行、记忆、感知、行动、情绪、反思和睡眠组合起来，跑几天或几周，看看会发生什么。

大公司通常不做这种实验，因为产品需要可预测；学术环境也不常做，因为开放式长期观察很难发表。但这个实验仍然值得有人做。
